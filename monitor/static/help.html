← Dashboard Metric Guide
This guide explains every metric shown on the dashboard. Your Mac Mini runs on Apple M4 — an Apple Silicon chip where CPU, GPU, and RAM all share the same unified memory pool. This means the GPU can use more RAM when needed (great for Ollama) but also means RAM pressure affects everything at once.
Compute
⚙️ CPU % 0 – 100%
How much of the 10-core M4 processor is in use right now. macOS averages across all cores. High CPU is normal for short bursts (compiling, video). Sustained high CPU with no obvious cause may indicate a runaway process — check the process table.
0–60% Normal 60–80% Busy 80%+ Hot
🎮 GPU % Apple M4 · 10 cores
How busy the integrated M4 GPU is. On Apple Silicon the GPU lives on the same chip as the CPU and uses shared (unified) memory.

Ollama runs AI models on the GPU — you'll see this spike when you send prompts. High GPU% + high GPU memory = a model is actively running inference.
0–40% Idle/light 40–80% Active 80%+ Heavy load
📊 Load Average 1m · 5m · 15m
The average number of processes waiting to run over the last 1, 5, and 15 minutes. On a 10-core chip, a load of 10.0 means every core is fully occupied. Values below 10 on this Mac are fine. Watch the 15m number — it tells you if load is trending up or down.
< 5 Normal 5–10 Busy > 10 Overloaded
Memory (Unified)
🧠 Memory % Used / Total GB
How much of the total unified memory (RAM) is in use. On Apple Silicon this pool is shared between CPU, GPU, and Neural Engine — so a large AI model loaded in Ollama will show here too.

macOS manages this aggressively: inactive pages are reclaimed automatically so high% isn't always bad — watch Memory Pressure instead.
🔴 Memory Pressure Normal · Moderate · Critical
Apple's own measure of memory health. More meaningful than raw % because it accounts for wired (pinned) and compressed memory.

Normal: plenty of free memory.
Moderate: macOS is compressing pages to make room — performance may start to dip.
Critical: macOS is swapping to disk — everything will feel sluggish.
Normal Moderate Critical
💾 GPU Memory GB currently used by GPU
How many GB of unified memory the GPU is actively using. When you load an Ollama model this number rises — a 7B model typically uses ~4–5 GB, a 13B model ~8–10 GB.

If GPU memory + system memory exceed the total (e.g. 24 GB), the Mac will start swapping, causing slow inference.
Ollama tip: Run ollama ps in Terminal to see which models are loaded and how much VRAM each is using.
Storage
💿 Disk % Used / Total GB
How full the main internal SSD is. macOS itself needs headroom for virtual memory (swap) — keep at least 10–15 GB free for smooth operation.

Ollama model files are stored on disk (usually in ~/.ollama/models). Each downloaded model can be 4–20 GB, so disk usage grows as you pull more models.
< 70% Fine 70–85% Getting full > 85% Action needed
Ollama AI Server
🤖 Ollama Status Running / Stopped
Whether the Ollama server is running at localhost:11434. When running, it shows any models currently loaded into GPU memory.

Running, no active models: Ollama is ready but idle — models may have been unloaded from GPU.
Running + model shown: A model is loaded and ready for inference (fast response to prompts).
Models stay loaded in GPU memory for ~5 minutes after last use, then auto-unload to free memory. The badge here shows what's currently hot in memory.
Performance Trend Chart
📈 Trend Chart 1h · 6h · 24h windows
Shows CPU%, RAM%, and GPU% over time. Data is sampled every 30 seconds and stored for up to 24 hours — history survives restarts.

1h — the default, best for watching real-time load (120 data points, 30s resolution).
6h — good for spotting recurring patterns (e.g., a scheduled backup).
24h — shows the full day, useful for spotting overnight load.
The chart refreshes every 30 seconds. GPU line appears as data accumulates — it will be absent on a fresh install until the first sample is written.
Processes
🔬 Top Processes Top 20 by CPU
The 20 processes using the most CPU right now. Tap any column header to re-sort by that column.

CPU % — current share of total CPU. Can briefly exceed 100% on multi-threaded work.
RAM MB — resident set size: physical memory this process is actually using right now.
Status — running means actively executing; sleeping is normal idle state.
Processes named kernel_task, WindowServer, and mds are normal macOS system processes. High kernel_task CPU is macOS throttling the chip due to thermals — try improving airflow.
Scheduled Jobs
⏰ Cron Jobs Unix scheduler
Traditional Unix scheduled tasks from your crontab. The 5 columns before the command are the schedule: minute hour day month weekday.

Example: 0 2 * * * = every day at 2:00 AM. An asterisk (*) means "every".
🚀 Launch Agents macOS scheduler
macOS-native background services managed by launchd — the recommended way to run things on a schedule or at login on a Mac.

At login — starts when you log in.
Every Xm/h — runs on a repeating interval.
HH:MM — runs at a specific time of day.
